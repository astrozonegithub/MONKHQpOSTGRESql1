# PostgreSQL Optimization for MonkHQ

Comprehensive PostgreSQL optimization configurations for both local Neo development and Koyeb production deployment.

## 📁 Files Overview

### Configuration Files
- **`neo_postgres.conf`** - Optimized configuration for local Neo development
- **`koyeb_postgres.conf`** - Production-optimized configuration for Koyeb deployment
- **`monitoring_dashboard.sql`** - SQL views and functions for database monitoring

### Setup Scripts
- **`setup_neo_postgres.sh`** - Automated setup script for local Neo environment
- **`setup_koyeb_postgres.sh`** - Automated setup script for Koyeb production
- **`koyeb_deploy.sh`** - Deployment script for Koyeb (generated by setup script)

### Django Settings
- **`django_neo_settings.py`** - Django database settings for Neo (generated)
- **`django_koyeb_settings.py`** - Django database settings for Koyeb (generated)

## 🚀 Quick Start

### Local Neo Development

```bash
# 1. Make the setup script executable
chmod +x postgres_optimization/setup_neo_postgres.sh

# 2. Run the setup script
./postgres_optimization/setup_neo_postgres.sh

# 3. Test the connection
psql -h localhost -U monkhq_user -d monkhq_neo
```

### Koyeb Production

```bash
# 1. Set Koyeb environment variables
export KOYEB_DB_HOST="your-koyeb-db-host"
export KOYEB_DB_PORT="5432"
export KOYEB_DB_NAME="monkhq_koyeb"
export KOYEB_DB_USER="monkhq_user"
export KOYEB_DB_PASSWORD="your-secure-password"

# 2. Run the setup script
chmod +x postgres_optimization/setup_koyeb_postgres.sh
./postgres_optimization/setup_koyeb_postgres.sh

# 3. Deploy to Koyeb
./postgres_optimization/koyeb_deploy.sh
```

## 🔍 Troubleshooting

### Common Issues

#### Connection Problems
```bash
# Check PostgreSQL status
sudo systemctl status postgresql

# Check logs
sudo tail -f /var/log/postgresql/postgresql-*.log

# Test connection
psql -h localhost -U monkhq_user -d monkhq_neo
```

#### Performance Issues
```sql
-- Check slow queries
SELECT * FROM monitoring.top_queries;

-- Check locks
SELECT * FROM pg_locks WHERE granted = false;

-- Check connections
SELECT * FROM monitoring.connection_activity;
```

#### Memory Issues
```sql
-- Check memory usage
SELECT * FROM monitoring.database_overview;

-- Check cache hit ratio
SELECT * FROM monitoring.database_overview WHERE metric = 'Cache Hit Ratio';
```
